About FoodHub

An End-to-End DevOps Showcase Portal

Project Overview

FoodHub is a state-of-the-art food ordering simulator designed for showing modern software delivery cycles. It mimics the front-end style of popular applications like Swiggy and Zomato, backed by a Spring Boot engine.

Technical Architecture

The core codebase is written in Java 17 leveraging the Spring Boot 3.x framework. It utilizes Spring Web, Spring Data JPA, H2 In-Memory database, and server-side Thymeleaf rendering engines. The application layout is fully responsive, leveraging Bootstrap 5 customized with custom CSS.

The DevOps Lifecycle

The repository is structured to run inside an automated deployment cycle:

Multi-Stage Dockerfile

Compiles with Maven and runs on a secure, light-weight Alpine JRE image to reduce attack vectors and minimize size.

Kubernetes Declarations

Employs LoadBalancer services and Deployment specs with memory resource limits and readiness probes to manage traffic.

Terraform Infrastructure

Spins up Virtual Networks, Resource Groups, Azure Container Registries, and Azure Kubernetes Service pools with system identities.

Azure Pipeline

Triggers unit testing, compiles dependencies, packs the Docker container, and logs it directly into Azure Container Registry.

Purpose of the Application

This application is designed specifically for sandbox/demonstration environments. It loads dummy data into H2 memory on startup and does not require complex database engines, making it portable and easy to run in container clusters.
